## M1 约定

返回结果给前端展示。

```go
type VaultInfo struct {
	Path string `json:"path"`
	Name string `json:"name"`
}
```

发现某个目录已经是 vault 后，默认不继续深入该 vault 内部扫描。

M1 不做深层递归扫描。用户选择的目录应满足以下形态之一，最多识别 2 层子目录：

```text
root/.obsidian
root/<vault>/.obsidian
root/<group>/<vault>/.obsidian
```

## M2 约定

M2 只做前端选择状态，不增加后端接口。

前端状态：

```text
mainVault
targetVaults
```

用户流程：

```text
扫描 vault
展示 vault 列表
点击一个 vault 作为主库
多选若干 vault 作为从库
进入 M3 设置同步文件
```

库列表持久化不属于 M2，后续作为优化项处理。

当前 UI 骨架：

```text
SyncWizard
扫描库 -> 选择库 -> 同步范围 -> 同步计划 -> 执行结果
```

当前已落地：

```text
扫描库
多次扫描会合并新发现的 vault，并按 path 去重
选择主库
选择多个从库
每次选择主库后，其他 vault 自动设为从库
主库不能同时作为从库
重新扫描不会清空已选择的主库和从库
```

库选择三态规则：

```text
主库：mainVault.path === vault.path
从库：targetVaults 包含 vault.path
未选中：既不是主库，也不在 targetVaults
```

交互规则：

```text
点击“主库”：当前 vault 设为主库，其他 vault 默认设为从库
点击“从库”：在从库和未选中之间切换
主库不能点成从库；如果要排除当前主库，必须先选择别的主库
```

相关前端文件：

```text

frontend/src/pages/scan_vaults.svelte
frontend/src/pages/steps/select_vaults_step.svelte
frontend/src/lib/components/vault_list.svelte

```

## M3 约定

同步文件使用相对 `.obsidian` 的路径，不使用绝对配置文件路径。

示例：

```text
appearance.json
hotkeys.json
plugins/dataview/
```

默认只勾选以下通用配置：

```text
app.json
core-plugins.json
hotkeys.json
appearance.json
snippets/
themes/
```

`appearance.json` 记录启用的 CSS 片段和主题，应与 `snippets/`、`themes/` 一起同步。

其他配置仍正常展示，但默认不勾选，允许用户手动勾选。

常见的社区插件列表、图谱、书签、属性类型和工作区配置提供用途及跳过原因说明。

默认规则只在用户首次进入同步范围设置时应用一次，后续允许用户编辑。

后端接口：


## M4 约定

`BuildSyncPlan` 只计算同步计划，不写文件，不创建备份目录。

计划需要展示：

```text
主库有哪些配置文件
哪些会复制到从库
哪些会跳过
是否存在危险操作
```

危险操作第一版只包含：镜像模式下删除从库多余配置文件。

MVP 第一版只实现覆盖模式，因此 M4 第一版不处理镜像模式危险操作。

## M5 约定

`ExecuteSyncPlan` 只接收 `SyncPlan`，不重新接收零散参数。

M5 第一版执行覆盖同步，不包含备份功能。

## 同步模式

### 覆盖模式

主库选中的配置文件覆盖到从库。

MVP 第一版只实现覆盖模式。

### 镜像模式

从库多余的配置文件也删除，使其和主库一致。

镜像模式暂不进入 MVP 第一版，后续实现前需要先完善危险操作确认。
