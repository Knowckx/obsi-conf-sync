## Windows 视口缩放状态

- 已定位 WebView2 `PutRasterizationScale(double)` 的 Win64 ABI 参数错误：必须传递 `math.Float64bits(scale)`，不能传递 `&scale`；修复已进入 Wails 主分支，但尚未包含在项目当前使用的版本中。
- 项目当前固定使用上游 Wails v3 `v3.0.0-alpha2.117`，`go.mod` 不再使用本地 Wails/WebView2 replacement。
- Wails v3 保留上游“仅 DPI 实际变化时同步”的逻辑，不在 `navigationCompleted()` 中无条件重同步或调用 `Resize()`。
- 2026-07-17 在 Windows 150% 缩放下单实例验证通过：冷启动、连续刷新、Vite HMR、最小化/恢复均保持 `dpr=1.5`、`innerWidth=1166`；跨不同 DPI 显示器拖动尚未验证。
- 详细根因、复现指标和后续多显示器定位方式见 `3.Wails_WebView2_视口缩放问题.md`。

| M3 | 设置同步文件 | 展示主库里的配置文件，用户选择需要同步的文件 | `ListConfigItems(vaultPath string) ([]ConfigItem, error)` | 同步路径使用相对 `.obsidian` 的路径；默认只勾选通用配置白名单，其余配置允许用户手动勾选 | 已验证 |

## M1 约定

返回结果给前端展示。

```go
type VaultInfo struct {
	Path string `json:"path"`
	Name string `json:"name"`
}
```

发现某个目录已经是 vault 后，默认不继续深入该 vault 内部扫描。

M1 不做深层递归扫描。用户选择的目录应满足以下形态之一，最多识别 2 层子目录：

```text
root/.obsidian
root/<vault>/.obsidian
root/<group>/<vault>/.obsidian
```

## M2 约定

M2 只做前端选择状态，不增加后端接口。

前端状态：

```text
mainVault
targetVaults
```

用户流程：

```text
扫描 vault
展示 vault 列表
点击一个 vault 作为主库
多选若干 vault 作为从库
进入 M3 设置同步文件
```

库列表持久化不属于 M2，后续作为优化项处理。

当前 UI 骨架：

```text
SyncWizard
扫描库 -> 选择库 -> 同步范围 -> 同步计划 -> 执行结果
```

当前已落地：

```text
扫描库
多次扫描会合并新发现的 vault，并按 path 去重
选择主库
选择多个从库
每次选择主库后，其他 vault 自动设为从库
主库不能同时作为从库
重新扫描不会清空已选择的主库和从库
```

库选择三态规则：

```text
主库：mainVault.path === vault.path
从库：targetVaults 包含 vault.path
未选中：既不是主库，也不在 targetVaults
```

交互规则：

```text
点击“主库”：当前 vault 设为主库，其他 vault 默认设为从库
点击“从库”：在从库和未选中之间切换
主库不能点成从库；如果要排除当前主库，必须先选择别的主库
```

相关前端文件：

```text

frontend/src/pages/steps/select_vaults_step.svelte
frontend/src/lib/components/vault_list.svelte
```


常见的社区插件列表、图谱、书签、属性类型和工作区配置提供用途及跳过原因说明。

## M4 约定

`BuildSyncPlan` 只计算同步计划，不写文件，不创建备份目录。

计划需要展示：

```text
主库路径
每个目标库将新增哪些配置
每个目标库将覆盖哪些配置
```

当前最小模型：

```go
type SyncRequest struct {
    MainVaultPath    string   `json:"mainVaultPath"`
    TargetVaultPaths []string `json:"targetVaultPaths"`
    SelectedPaths    []string `json:"selectedPaths"`
}

type SyncPlan struct {
    MainVaultPath string           `json:"mainVaultPath"`
    Targets       []TargetSyncPlan `json:"targets"`
}

type TargetSyncPlan struct {
    VaultPath string   `json:"vaultPath"`
    Create    []string `json:"create"`
    Overwrite []string `json:"overwrite"`
}
```

当前实现文件：

```text
go_src/inner/svc/sync_models.go
go_src/inner/svc/sync_plan.go
frontend/src/lib/api/vault_service.ts
frontend/src/pages/steps/sync_plan_step.svelte
```

前端计划页在目标库列表前显示简洁的主库来源卡片；目标库卡片只突出“新增配置”和“覆盖现有配置”，不再展示跳过项。

危险操作第一版只包含：镜像模式下删除从库多余配置文件。

MVP 第一版只实现覆盖模式，因此 M4 第一版不处理镜像模式危险操作。

## M5 约定

`ExecuteSyncPlan` 只接收 `SyncPlan`，不重新接收零散参数。

M5 第一版执行覆盖同步，不包含备份功能。

当前实现：

```text
后端：
go_src/inner/svc/sync_models.go
go_src/inner/svc/sync_execute.go
go_src/inner/svc/vaultservice.go

前端：
frontend/src/lib/api/vault_service.ts
frontend/src/pages/sync_wizard.svelte
frontend/src/pages/steps/sync_result_step.svelte
```

执行规则：

```text
文件和目录均支持复制
目录复制采用递归覆盖，不删除目标目录中的额外文件
执行中通过原子状态拒绝新的 ExecuteSyncPlan 调用，防止前端抖动造成并发重复执行
单个目标库的新增、覆盖和失败项分别记录在 SyncResult 中
前端确认同步后进入结果页，展示总体及各目标库结果
结果页的成功项使用 ✅、失败项使用 ❌，同时保留状态分组和错误文字
结果页不提供“上一步”，仅以“完成”结束流程；点击后保留已扫描的库列表，清空本轮同步状态并返回扫描库步骤
```

## 同步模式

### 覆盖模式

主库选中的配置文件覆盖到从库。

MVP 第一版只实现覆盖模式。

### 镜像模式

从库多余的配置文件也删除，使其和主库一致。

镜像模式暂不进入 MVP 第一版，后续实现前需要先完善危险操作确认。
