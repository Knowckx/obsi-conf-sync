


### 前端页面的布局结构
- 本文只说明页面结构与各层职责。
- 示例中的 Tailwind 类仅用于直观表达布局效果；实际实现应使用语义类名和 scoped CSS

#### 常见结构

```
App
 └── AppLayout                // 应用级布局骨架
      ├── Header              // 顶部导航
      ├── Sidebar             // 侧边栏
      ├── Footer              // 
      └── Page                 // 页面 路由出口
            ├── Section        // 页面区域
            │    └── Card/Component
            └── ...
```

AppLayout → Page → Section → Card

#### 1.AppLayout 应用骨架
确立整页的基调。 整个应用的骨架。
```
<AppLayout class="min-h-screen bg-slate-50 text-slate-900 flex flex-col">
    <!-- 包含 Header, Sidebar, Footer, Content -->
</AppLayout>    
```
页面高度至少占满屏幕
全局底色
默认文字

#### 2.Page 页面
Page 使用单层 `<main>`，负责页面语义、最大宽度、水平居中和页面内边距。  
<main> 是 HTML 语义标签，表示当前页面的主体内容。

```html
<main class="box-border w-full max-w-6xl mx-auto px-6 py-10">
  <!-- 页面核心内容 -->
</main>
w-full 让 Page 宽度占满父容器
max-width 最大宽度用于防止内容在大屏上被无限拉伸。
mx-auto 水平居中，并通过内边距避免内容紧贴窗口边缘。
只有出现实际响应式需求时，才增加不同断点的内边距。
```

#### 3. Section 页面区块
使用 `<section>` 将页面内容按业务职责划分为独立区块。

Section 默认使用单列 Grid 排列内部内容，并管理直属内容之间的间距：

```html
<section class="grid gap-4">
  <!-- 放置当前区块的业务内容或 Card -->
</section>
```
不同 Section 之间的间距由 Page 负责。
列数和响应式布局属于具体业务需求，出现实际需求后在业务组件中定义。


#### 4.Card 卡片
Card 是 Section 内具有明确视觉边界的内容容器。

Card 使用纵向 Flex 排列内部内容，并负责直属内容之间的间距、内边距、背景、边框、圆角和轻阴影：

```html
<div class="flex flex-col gap-4 rounded-lg border border-slate-200 bg-white p-5 shadow-sm">
  <!-- 卡片内部内容 -->
</div>
```

Card 不负责自身与其他组件之间的外部间距，外部间距由所在的 Section 或其他父容器管理。
