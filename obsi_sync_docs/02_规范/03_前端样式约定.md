# 前端样式约定

> 文档更新时间 2026-07-12

## 样式分层

前端样式按以下层级维护：

1. `frontend/src/app.css`：全局基础样式和设计变量。
2. `infa-s5`：按钮、输入框、卡片等通用 UI 组件。
3. `frontend/src/lib/components/`：项目内可复用且具有独立职责的组件。
4. Svelte 组件 `<style>`：仅维护当前组件专属的结构和布局。

禁止为了集中管理而把所有页面样式移入全局文件。

## 全局样式

`app.css` 只维护：

- reset 和应用根节点样式。
- 字体等应用级基础设置。
- 颜色、间距、圆角等通用 CSS 变量。
- 确实跨多个页面复用的少量语义样式。

通用视觉值应先定义为 CSS 变量，页面中通过 `var(...)` 使用。禁止在多个组件中重复硬编码相同颜色、间距或圆角。

CSS 变量按语义命名，例如：

```css
:root {
  --color-text: #344054;
  --color-text-muted: #667085;
  --color-border: #d0d5dd;
  --color-danger: #b42318;
  --radius-control: 6px;
}
```

变量只在出现实际复用时增加，不预先建设完整设计系统。

## 组件样式

- 按钮、输入框、卡片等已有能力优先使用 `infa-s5`，不重复实现。
- 稳定重复且有独立职责的 UI 结构才提取为项目公共组件。
- 单薄样式包装、只改类名的组件不提取。
- 公共组件负责自身视觉和交互状态，页面不应依赖其内部 DOM 结构覆盖样式。

## 页面样式

Svelte 组件默认使用 scoped `<style>`，用于维护：

- 页面或组件专属布局。
- 当前组件内部的排列、尺寸和响应式变化。
- 只在当前业务语义下存在的视觉状态。

页面样式应使用全局设计变量，不直接定义项目通用视觉值。

禁止：

- 使用 `:global(...)` 修改其他组件内部样式。
- 通过复杂选择器依赖深层 DOM 结构。
- 为一次性场景增加全局类。
- 未形成复用前创建样式工具层或主题系统。

## 布局与响应式

- 优先使用 Grid 和 Flexbox。
- 页面布局优先从窄屏自然扩展到宽屏。
- 相同布局场景使用一致的间距和断点。
- 新增断点前先复用现有断点；确有不同布局需求时再增加。
- 响应式规则保留在对应组件 `<style>` 中。

## 样式技术边界

- Tailwind 用于 `infa-s5` 的构建和既有组件能力，不要求业务页面改写为工具类风格。
- 业务页面默认使用语义类名和 scoped CSS。
- 不在同一组件中混用大量 Tailwind 工具类与重复的 scoped CSS。
- 未经明确需求，不引入新的 CSS 框架、预处理器或 CSS-in-JS 方案。

## 命名

- 类名使用小写短横线，例如 `.config-list`、`.vault-context`。
- 类名表达业务或结构语义，不使用颜色、位置等偶然表现命名。
- 状态类使用简短语义，例如 `.active`、`.error`、`.muted`。
