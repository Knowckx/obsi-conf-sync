# 版本跟踪

| 版本 | 阶段 | 需求 | 后端接口 | 核心规则 | 状态 |
| --- | --- | --- | --- | --- | --- |
| M1 | 扫描可用 | 选择目录；识别当前目录或一级子目录中的 vault；展示列表 | `ScanVaults(root string) ([]VaultInfo, error)` | `root/.obsidian/` 存在时 root 是 vault；否则只检查 `root/*/.obsidian/`；一级子目录跳过隐藏目录、`node_modules`、`.git` | 已验证 |

## M1 说明

返回结果给前端展示。建议返回字段：

```go
type VaultInfo struct {
	Path string `json:"path"`
	Name string `json:"name"`
}
```

发现某个目录已经是 vault 后，默认不继续深入该 vault 内部扫描。

M1 不做深层递归扫描。用户选择的目录应满足以下形态之一：

```text
root/.obsidian
root/<vault>/.obsidian
```
