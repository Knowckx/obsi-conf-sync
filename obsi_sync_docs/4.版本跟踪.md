# 版本跟踪

| 版本 | 阶段 | 需求 | 后端接口 | 核心规则 | 状态 |
| --- | --- | --- | --- | --- | --- |
| M1 | 扫描可用 | 选择目录；识别当前目录或最多 2 层子目录中的 vault；展示列表 | `ScanVaults(root string) ([]VaultInfo, error)` | `root/.obsidian/` 存在时 root 是 vault；否则最多检查到 `root/*/*/.obsidian/`；扫描子目录时跳过隐藏目录、`node_modules`、`.git` | 已验证 |
| M2 | 库列表持久化 | 用户可把扫描到的有效库追加到已保存库列表；重启后仍保留；支持移除 | `ListSavedVaults() ([]VaultInfo, error)`；`AddSavedVaults(vaults []VaultInfo) ([]VaultInfo, error)`；`RemoveSavedVault(path string) ([]VaultInfo, error)` | `ScanVaults` 只负责发现 vault；保存列表单独管理；按规范化后的 `Path` 去重；配置写入用户配置目录 | 待开发 |

## M1 说明

返回结果给前端展示。建议返回字段：

```go
type VaultInfo struct {
	Path string `json:"path"`
	Name string `json:"name"`
}
```

发现某个目录已经是 vault 后，默认不继续深入该 vault 内部扫描。

M1 不做深层递归扫描。用户选择的目录应满足以下形态之一，最多识别 2 层子目录：

```text
root/.obsidian
root/<vault>/.obsidian
root/<group>/<vault>/.obsidian
```

## M2 说明

M2 负责记住用户确认保存的 vault 列表，不改变 M1 的扫描职责。

用户流程：

```text
选择目录并扫描
展示本次发现的 vault
用户确认添加到已保存库列表
用户可继续选择其他目录追加
重启软件后仍能看到已保存库列表
```

保存规则：

```text
按 Path 去重
保存前做 filepath.Abs / filepath.Clean
配置写到 os.UserConfigDir() 下的 obsi-conf-sync/config.json
```
