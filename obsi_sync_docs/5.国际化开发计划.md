## 第一阶段：泛用模块

状态：进行中。

预计工作量：3 至 5 小时。

- [ ] 建立 `frontend/src/lib/i18n/core/`。
- [ ] 将工厂、类型和插值分别放入 `create_i18n.svelte.ts`、`types.ts`、`interpolate.ts`。
- [ ] 使用泛型 `CreateI18nOptions<T, TDefaultLocale>` 保留项目文字表的字面量类型。
- [ ] 从 `T` 推导 Locale，从 `T[TDefaultLocale]` 推导 MessageKey，禁止退化为 `string`。
- [ ] 拆分类型基准语言 `defaultLocale` 和运行时回退语言 `fallbackLocale`。
- [ ] 约束 `defaultLocale`、`fallbackLocale` 和 `htmlLang` 的语言键为 `keyof T`。
- [ ] 实现“当前语言 → fallbackLocale → 原始 key”的文案查找顺序。
- [ ] 实现基于 `Record<string, string | number>` 的轻量参数插值。
- [ ] 将系统语言匹配改为基于项目支持语言。
- [ ] 将存储键、fallback 和 HTML 映射改为配置项。
- [ ] 在实例初始化和 `setLocale()` 时自动同步 `<html lang>`。
- [ ] 将现有语言状态和持久化原型接入最终工厂。
- [ ] 更新本项目 `index.svelte.ts` 实例配置。

完成标准：

- 泛用能力全部位于 `i18n/core/`，不包含项目文案和固定语言。
- `i18n.t()` 只接受基准语言中存在的翻译 key。
- 无效语言配置在 TypeScript 检查阶段报错。
- 缺失翻译、参数插值和 HTML lang 同步符合设计文档。
