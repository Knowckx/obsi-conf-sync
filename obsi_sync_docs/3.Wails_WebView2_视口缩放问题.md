# Wails WebView2 视口缩放问题

> 最近更新：2026-07-17

## 问题与根因

Windows 150% 缩放下，页面曾在刷新后明显变小：

```text
异常：devicePixelRatio=0.5，innerWidth=3498
正常：devicePixelRatio=1.5，innerWidth=1166
```

问题不在 CSS。旧版 `webview2 v1.0.24` 将 COM 的 `PutRasterizationScale(double)` 参数错误地按指针传递：

```go
uintptr(unsafe.Pointer(&scale))
```

`double` 应按值并遵循目标架构 ABI 传递。错误参数会使 WebView2 获得异常光栅化比例，进而放大 DOM 视口。

## 上游状态

Wails 已分两层修复：

- [#5572](https://github.com/wailsapp/wails/pull/5572)：在窗口恢复和 DPI 变化时同步缩放。
- [#5734](https://github.com/wailsapp/wails/pull/5734)、[#5761](https://github.com/wailsapp/wails/pull/5761)：按托管模式确定唯一的缩放管理方，避免 WebView2 与宿主同时写入比例。
- [d8320ef](https://github.com/wailsapp/wails/commit/d8320ef8cf9b6eedd308e0b84b1693878ad24294)：修复 `PutRasterizationScale` 及同类 COM 参数的 ABI 传递。
- [#5711](https://github.com/wailsapp/wails/pull/5711)：将 WebView2 封装合并到 `v3/internal/webview2`，后续随 Wails v3 一起发布。

`v3.0.0-alpha2.117` 已包含缩放管理策略修复，但仍依赖旧的 `webview2 v1.0.24`；ABI 修复目前已进入 Wails 主分支，等待后续版本标签发布。

## 当前项目状态

项目已直接升级为：

```text
github.com/wailsapp/wails/v3 v3.0.0-alpha2.117
```

`go.mod` 已删除本地 Wails/WebView2 replacement，不再使用 `../wails-v3-local`。

2026-07-17 在 Windows 150% 缩放、单实例下验证：

```text
冷启动：dpr=1.5，innerWidth=1166，innerHeight=853
连续刷新 3 次：指标保持不变
最小化/恢复：指标保持不变
html/body zoom=1
```

原问题在当前环境下未复现。尚未验证跨不同 DPI 显示器拖动。

## 后续处理

- 当前固定使用 `v3.0.0-alpha2.117`，不跟踪 Wails 主分支。
- 后续升级到包含 `d8320ef` 的版本标签，并确认不再依赖独立的 `webview2 v1.0.24`。
- 如问题再次出现，只保留一个 Wails 实例，记录 `devicePixelRatio`、窗口尺寸、`visualViewport` 和 WebView2 DPI 日志。
- 不使用 CSS `zoom`、固定 Wails `Zoom` 或 `--force-device-scale-factor` 掩盖问题。
